# Git-Eval A ランク: Architect

設計判断ができるかを見るランクです。LLM 中心の評価になります。

## お題: 設計判断を伴うプロジェクトを作ろう

技術選定の理由を明確にし、設計の妥当性が問われます。

### やること

1. プロジェクトを作る（または既存プロジェクトに YAML を追加）
2. `git-eval-config.json` を設定する
3. `DECISION.md` に設計判断を書く（推奨）
4. テスト・リンターを通す
5. commit して push する

### DECISION.md に書くべきこと

- なぜこの技術/ライブラリを選んだか
- 何を想定してこの設計にしたか
- なぜ機能をあえて削った/シンプルにしたか
- 既知の妥協点とその理由

### 評価ポイント（100 点満点）

| チェック項目 | 配点 | 評価方法 |
|---|---|---|
| テスト・リンター通過 | 10 | CI/CD |
| CRLF / .env チェック | 10 | CI/CD |

| 言語イディオムの活用 | 15 | LLM |
| トレードオフの根拠 | 20 | LLM |
| セキュリティへの配慮 | 10 | LLM |
| DECISION.md の質 | 10 | LLM |
